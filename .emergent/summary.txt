original_problem_statement:
The user wants to build a web-based SaaS platform named Okul Öncesi Denetim ve Raporlama Sistemi (Preschool Inspection and Reporting System). The platform's goal is to help private preschools prepare for official inspections (MEB, Fire, Agriculture) by identifying deficiencies and generating reports.

**PRODUCT REQUIREMENTS:**

The system consists of three main modules:
1.  **Website (Marketing & Sales):**
    *   Pages: Homepage, Services, Packages, News, Contact.
    *   Goal: Drive preschool owners to purchase an inspection service.
    *   Features: Strong CTAs, package purchasing flow, contact forms.
2.  **Admin Panel:**
    *   A comprehensive backend for managing the entire operation.
    *   Features: User/Staff management, CRM for contact forms, package management, audit category/question management, service area management, payment viewing, and a reporting dashboard to view/manage audits.
3.  **Field Staff Tablet Interface:**
    *   An interface for inspectors to use on-site.
    *   Features: View assigned audits, navigate through audit questions, select answers, add notes, capture photos, and finalize inspections.

**Key Technology & Flows:**
*   **Tech Stack:** Next.js, Tailwind, Node.js, PostgreSQL (currently SQLite).
*   **Authentication:** Phone number + password.
*   **Payments:** ParamPOS integration (currently mocked).
*   **Reporting:** Server-side PDF generation.
*   **Notifications:** SMS via NetGSM (currently mocked).

**User's preferred language**: Turkish (Türkçe)
